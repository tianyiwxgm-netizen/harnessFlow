## Review P1/P2/P3 修补决策

### P1-01 · FALLBACK_RAW 阈值 (方案 1 · 代码对齐 3-1)

- **设计 3-1 L2-05 §5.3 / §6.4 timeline3**:
  - 1-2 signals fail → SKIP_SIGNAL (redistribute weights)
  - ≥ 3 signals fail → FALLBACK_RAW (raw recall order + top_k cap)
- **旧代码**：`len(signals_skipped) >= 5` · 只有全失败才 fallback
- **修后代码** (`service.py:188`):
  ```python
  if len(signals_skipped) >= 3:
      return self._fallback_raw(req, top_k, top_k_capped, warnings, signals_skipped)
  ```
- **error_code 二档区分** (对齐 §11 表):
  - skipped == 5 → `ALL_SCORERS_FAILED` (严重 bug)
  - 3 ≤ skipped < 5 → `SCORE_COMPUTE_FAIL` (3+ accumulating · §2091)

### P2-02 · degradation_level gate 补实装

- 旧代码：`_degradation_level` 在 `_try_emit` 里被设 `READ_ONLY_ISOLATION` / `EMERGENCY_LOCKDOWN`，但 `resolve_read_scope` / `allocate_session_write_slot` 不检查
- **新 gate**:
  - `resolve_read_scope`: `EMERGENCY_LOCKDOWN` → 立即 deny E-TIER-011 (READ_ONLY 仍允许读 · 按名保留)
  - `allocate_session_write_slot`: 任一降级档 → 立即 deny E-TIER-011

### P2-03 · kind_priority 动态 step

- 旧: `return max(0.0, 1.0 - 0.1 * idx)` (hardcoded 10-item assumption)
- 新: `step = 1.0 / max(len(priority_list), 1); return max(0.0, 1.0 - step * idx)`
- 2-item list 现在 ranks = 1.0, 0.5 (之前 1.0, 0.9)

### P2-04 · idem_cache (project_id, rerank_id) 双元组 key

- 旧: `dict[str, RerankResponse]` keyed on `rerank_id`
- 新: `dict[tuple[str, str], RerankResponse]` keyed on `(project_id, rerank_id)`
- 消除同 UUID 跨 project 的缓存污染可能 (PM-14 加固)

### P3-01 · _scorers.py 覆盖率 67.6% → 96.4%

- 36 个 unit TC 覆盖 `_jaccard` / `_parse_iso` / 5 scorer 边缘分支
